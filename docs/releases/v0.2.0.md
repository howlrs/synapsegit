# SynapseGit v0.2.0 — Stage 0 preview

`v0.2.0` is the second tagged SynapseGit preview. It adds a bounded browser
import and Human-review slice to the existing local creator Pilot while keeping
the Core v0.1 protocol, `sg-oid-v1`, and localhost `/api/v1` route unchanged.
It is a source-available evaluation release, not a production release.

## Highlights

- `synapse-local` can stage an original, current, and caller-supplied candidate
  file from the browser, publish an AI-attributed proposal, and record Human
  `adopt`, `reject`, or `defer` in the same running process.
- Creator writes use bounded repository verification, prospective graph and
  Ref capacity checks, per-project mutation serialization, and durable
  committed receipts. A successful publication is not retried when rebuilding
  the full report reaches a resource limit.
- Release and CI automation now validate a versioned GitHub security baseline,
  including protected linear-history pull-request merges and immutable `v*`
  tags.
- Pinned GitHub Actions and direct Rust dependencies were refreshed. Protocol
  fixture OIDs remain unchanged, and bundled SQLite stays at the newest version
  compatible with the Rust 1.88 policy.
- The archive now bundles `LICENSE`, `THIRD_PARTY_NOTICES.md`, `SECURITY.md`,
  and `CHANGELOG.md`, and receives a GitHub build-provenance attestation.

## Install on Linux x86-64 GNU

The prebuilt archive is produced on Ubuntu 22.04 and requires glibc 2.34 or
newer. Other supported Unix-like hosts can build from the fixed tag with Rust
1.88 or newer.

```bash
curl -LO https://github.com/howlrs/synapsegit/releases/download/v0.2.0/synapsegit-v0.2.0-x86_64-unknown-linux-gnu.tar.gz
curl -LO https://github.com/howlrs/synapsegit/releases/download/v0.2.0/SHA256SUMS
sha256sum --check SHA256SUMS

gh attestation verify synapsegit-v0.2.0-x86_64-unknown-linux-gnu.tar.gz \
  --repo howlrs/synapsegit \
  --signer-workflow howlrs/synapsegit/.github/workflows/release.yml \
  --source-ref refs/tags/v0.2.0 \
  --deny-self-hosted-runners

tar -xzf synapsegit-v0.2.0-x86_64-unknown-linux-gnu.tar.gz
./synapsegit-v0.2.0-x86_64-unknown-linux-gnu/synapse --version
./synapsegit-v0.2.0-x86_64-unknown-linux-gnu/synapse-local --version
```

Both binaries should report `0.2.0`. See the
[installation guide](https://github.com/howlrs/synapsegit/blob/v0.2.0/docs/install.md)
for source installation and update precautions.

## Browser write boundary

Run `synapse-local` as a native single-user process and open only the exact
`http://127.0.0.1:...` URL it prints. Do not expose it through a reverse proxy.
The browser token is process-local authentication, not real-user or multi-user
authentication.

Each imported file is limited to 64 MiB and the three files to 192 MiB total.
At most two uploads stage concurrently, eight pending reviews are retained per
project, and 64 are retained per process. Review authority is intentionally
memory-only: restarting before a decision leaves an incomplete proposal that
cannot be resumed. Do not run another writer against a repository while
`synapse-local` owns it.

The third file is caller-supplied. SynapseGit does not invoke an AI model,
inspect pixels or EXIF, decide whether two images look alike, prove authorship,
or establish a physical change. Its comparison is primary-Blob byte identity
with partial comparability.

## Known limits

- Core v0.1, the OID profile, and archive format remain Stage 0 drafts without
  a stable compatibility promise.
- Browser `fsck`, archive export/restore, incomplete-session diagnostics, and
  restart-durable review authority are not implemented; use the CLI for the
  implemented maintenance operations.
- Capture, pixel registration, visual or physical change interpretation,
  model invocation, HTTP/JWT/MFA, and a production multi-user service are not
  implemented.
- The prebuilt artifact is Linux x86_64 GNU only. No crates.io, GHCR, Windows,
  macOS, or Linux ARM64 package is published.

No intentional Core v0.1 object, OID, or archive-format change was made since
v0.1.0, but important repositories should still be exported with the old
binary before updating. Read the [changelog](https://github.com/howlrs/synapsegit/blob/v0.2.0/CHANGELOG.md),
[security model](https://github.com/howlrs/synapsegit/blob/v0.2.0/docs/security_model.md),
and [localhost runbook](https://github.com/howlrs/synapsegit/blob/v0.2.0/deploy/local/README.md)
before evaluating important data.
