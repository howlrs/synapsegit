# SynapseGit v0.4.0 — Stage 0 preview

`v0.4.0` is the fourth tagged SynapseGit preview. It freezes the
provider-neutral generic-artifact v1 application contract and adds source-level
Rust libraries for bounded mapping, sequential Human review, restart
reconciliation, selected-site checkout, and deterministic local public bundles.
It also includes the multi-session publication-verification optimization and a
frozen publication-comprehension corpus. The Core v0.1 protocol, `sg-oid-v1`,
directory archive, and existing command surfaces have no intentional format
change. This remains a source-available evaluation release, not a production or
hosted service.

## Highlights

The generic-artifact items below describe tagged source/workspace libraries,
not new commands or routes in the packaged binaries.

- `synapsegit.generic-artifact` v1 now has committed schemas, fixtures, digest
  vectors, canonical timestamps, and exact scaled-integer semantics without
  floating-point values.
- `synapse-artifact` validates an entire bounded manifest before its first CAS
  write, rejects unsafe or colliding portable paths, and creates deterministic
  nested ManifestTrees without updating a Ref. Initial and sequential Proposal
  workflows preserve prior attempts, allow only one active review, and reverify
  the exact accepted base before a later Proposal.
- Human Decisions require a fresh host-authenticated, project-authorized,
  expiring one-shot approval bound to the exact actor, session, project epoch,
  Proposal, expected Decision head, disposition, and private rationale.
- `synapse-artifact-journal` and the durable application orchestrator record
  Proposal and Decision intents, reconcile bounded live state after restart,
  and require fresh authentication and authorization. The journal does not
  restore credentials, approvals, authority, handles, or permits.
- Selected-site checkout is bounded and revalidated. The generic-artifact
  projection and renderer produce deterministic, local-only Synapse or GitHub
  staging bundles without Git, network, upload, or credential operations.
- Multi-session image publication shares one snapshot-scoped bounded `fsck` and
  one disposable projection rebuild while retaining independent per-session
  lineage validation.
- A frozen publication-comprehension corpus covers complete adopt, reject, and
  defer outcomes plus incomplete-only views. Its executable scorer records
  external Human, AI, and accessibility evaluation as `not_run` until those
  evaluations are actually performed.

## Install on Linux x86-64 GNU

The prebuilt archive is produced on Ubuntu 22.04 and requires glibc 2.34 or
newer. Other supported Unix-like hosts can build from the fixed tag with Rust
1.88 or newer.

```bash
curl -LO https://github.com/howlrs/synapsegit/releases/download/v0.4.0/synapsegit-v0.4.0-x86_64-unknown-linux-gnu.tar.gz
curl -LO https://github.com/howlrs/synapsegit/releases/download/v0.4.0/SHA256SUMS
sha256sum --check SHA256SUMS

gh attestation verify synapsegit-v0.4.0-x86_64-unknown-linux-gnu.tar.gz \
  --repo howlrs/synapsegit \
  --signer-workflow howlrs/synapsegit/.github/workflows/release.yml \
  --source-ref refs/tags/v0.4.0 \
  --deny-self-hosted-runners
```

Stop if either verification command fails. Do not extract or install an
unverified archive.

```bash
tar -xzf synapsegit-v0.4.0-x86_64-unknown-linux-gnu.tar.gz

mkdir -p "$HOME/.local/bin"
install -m 0755 synapsegit-v0.4.0-x86_64-unknown-linux-gnu/synapse "$HOME/.local/bin/synapse"
install -m 0755 synapsegit-v0.4.0-x86_64-unknown-linux-gnu/synapse-local "$HOME/.local/bin/synapse-local"
install -m 0755 synapsegit-v0.4.0-x86_64-unknown-linux-gnu/synapse-present "$HOME/.local/bin/synapse-present"
export PATH="$HOME/.local/bin:$PATH"

synapse --version
synapse-local --version
synapse-present --version
```

All three binaries should report `0.4.0`. See the
[installation guide](https://github.com/howlrs/synapsegit/blob/v0.4.0/docs/install.md)
for source installation and update precautions.

## Source-library boundary

The tagged source contains the generic-artifact workflow, schema, journal,
checkout, and local projection libraries. Workspace crates remain excluded from
crates.io publication. Evaluators can exercise the implementation from the
fixed source tag:

```bash
git clone --branch v0.4.0 --depth 1 https://github.com/howlrs/synapsegit.git
cd synapsegit
cargo test --workspace --all-targets --locked
```

The release archive deliberately remains the same three-binary surface:
`synapse`, `synapse-local`, and `synapse-present`. It does not add a
generic-artifact command, HTTP route, browser workflow, remote publisher, or
fourth binary. Existing image-oriented UI and CLI surfaces do not expose these
new source libraries.

## Compatibility and migration

- Repositories created with v0.3.0 require no planned format migration. Core
  v0.1 objects, `sg-oid-v1`, and directory archive semantics are unchanged.
- The public Rust `ApplicationError` enum adds `StaleBase` (`stale_base`) and
  `RefConflict` (`ref_conflict`). Downstream exhaustive matches must handle the
  new variants.
- Evaluation databases created by earlier untagged generic-artifact journal
  schema-v1 builds are transactionally upgraded in place to v2 when opened.
  They cannot then be reopened by those schema-v1 builds, so preserve a copy
  before evaluating the migration. Legacy rows are reconciled conservatively
  and are never promoted into fresh authority.

Stage 0 APIs and formats do not yet carry a stable compatibility promise.
Export important repositories with the old binary before updating so that a
known-good recovery copy remains available.

## Known limits

- Generic-artifact support is source-library-only in this release. There is no
  generic-artifact HTTP, CLI, browser, or remote publication surface.
- V1 accepts caller-supplied AI-attributed bytes and always records execution as
  unverified. It cannot represent a trusted model executor.
- Credentials and approvals remain process-local. Recovery requires fresh
  authentication and authorization, and never reconstructs prior authority.
- The artifact journal is a separate private SQLite store and is not atomic
  with the Core Ref store; recovery therefore reconciles explicit durable
  intents against bounded live state.
- Generic public bundles are unsigned local derived views, not proofs of
  authorship, truth, permission, completeness, or physical change.
- The existing browser and command surfaces remain image-workflow-oriented.
- The prebuilt artifact is Linux x86_64 GNU only. No crates.io, GHCR, Windows,
  macOS, or Linux ARM64 package is published.
