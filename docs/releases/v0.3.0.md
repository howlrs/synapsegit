# SynapseGit v0.3.0 — Stage 0 preview

`v0.3.0` is the third tagged SynapseGit preview. It adds a read-only publication
companion for explaining local history to people and AI, plus creator-session
diagnostics and explicitly confirmed background `fsck` in the loopback-only
application. The Core v0.1 protocol, `sg-oid-v1`, archive semantics, and existing
`synapse` commands remain unchanged. This is a source-available evaluation
release, not a production or hosted service.

## Highlights

- The archive now contains three binaries: `synapse`, `synapse-local`, and the
  new `synapse-present` companion.
- `synapse-present` derives a provider-neutral public projection and renders a
  deterministic local bundle for the default Synapse target or explicit
  `--synapse` / `--github` target. It performs no upload or network access.
- Public bundles contain canonical JSON, escaped Markdown, JavaScript-free HTML,
  a manifest, checksums, and a target-specific layout. Private rationale,
  internal Actor IDs, repository paths, and raw assets are omitted.
- `synapse-local` can diagnose incomplete creator sessions without rebuilding
  review authority or mutating history, and can run a server-bounded `fsck`
  after the user confirms the exact project key.
- Read-only publication opens the existing CAS without mutation and reads Ref
  state from a bounded private SQLite copy. Sidecars or concurrent source
  changes fail closed.

## Install on Linux x86-64 GNU

The prebuilt archive is produced on Ubuntu 22.04 and requires glibc 2.34 or
newer. Other supported Unix-like hosts can build from the fixed tag with Rust
1.88 or newer.

```bash
curl -LO https://github.com/howlrs/synapsegit/releases/download/v0.3.0/synapsegit-v0.3.0-x86_64-unknown-linux-gnu.tar.gz
curl -LO https://github.com/howlrs/synapsegit/releases/download/v0.3.0/SHA256SUMS
sha256sum --check SHA256SUMS

gh attestation verify synapsegit-v0.3.0-x86_64-unknown-linux-gnu.tar.gz \
  --repo howlrs/synapsegit \
  --signer-workflow howlrs/synapsegit/.github/workflows/release.yml \
  --source-ref refs/tags/v0.3.0 \
  --deny-self-hosted-runners

tar -xzf synapsegit-v0.3.0-x86_64-unknown-linux-gnu.tar.gz

mkdir -p "$HOME/.local/bin"
install -m 0755 synapsegit-v0.3.0-x86_64-unknown-linux-gnu/synapse "$HOME/.local/bin/synapse"
install -m 0755 synapsegit-v0.3.0-x86_64-unknown-linux-gnu/synapse-local "$HOME/.local/bin/synapse-local"
install -m 0755 synapsegit-v0.3.0-x86_64-unknown-linux-gnu/synapse-present "$HOME/.local/bin/synapse-present"
export PATH="$HOME/.local/bin:$PATH"

synapse --version
synapse-local --version
synapse-present --version
```

All three binaries should report `0.3.0`. See the
[installation guide](https://github.com/howlrs/synapsegit/blob/v0.3.0/docs/install.md)
for source installation and update precautions.

## Generate a local public view

After creating a local creator session with `synapse`, generate a GitHub-ready
view without publishing it:

```bash
synapse-present export /path/to/repository ./public-view \
  --session SESSION_ID \
  --public \
  --github

synapse-present preview ./public-view
```

Synapse output is the default; `--synapse` makes that selection explicit.
`--github` changes only the local target layout. Neither option authenticates to
or contacts a remote service. Inspect the generated files before choosing any
separate publication mechanism.

## Local application boundary

Run `synapse-local` as a native single-user process and open only the exact
`http://127.0.0.1:...` URL it prints. Do not expose it through a reverse proxy.
The browser token is process-local authentication, not real-user or multi-user
authentication.

Browser `fsck` uses fixed service limits and process-local job state. It does
not export, restore, repair, resume, or clean up history. Review authority is
also memory-only: restarting before a decision leaves an incomplete proposal
that diagnostics can explain but cannot resume.

## Known limits

- Core v0.1, the OID profile, archive format, and public Rust APIs remain Stage
  0 drafts without a stable compatibility promise.
- Publication bundles are unsigned derived views, not proofs of authorship,
  truth, permission, source completeness, or physical change.
- Raw-asset rendering, thumbnails, remote GitHub/Synapse publishing,
  credentials, destination diffs, and publication receipts are not implemented.
- Capture, pixel registration, visual or physical change interpretation, model
  invocation, HTTP/JWT/MFA, durable authorization, and a production multi-user
  service are not implemented.
- The prebuilt artifact is Linux x86_64 GNU only. No crates.io, GHCR, Windows,
  macOS, or Linux ARM64 package is published.

No intentional Core v0.1 object, OID, or archive-format change was made since
v0.2.0, but important repositories should still be exported with the old binary
before updating. Read the
[changelog](https://github.com/howlrs/synapsegit/blob/v0.3.0/CHANGELOG.md),
[security model](https://github.com/howlrs/synapsegit/blob/v0.3.0/docs/security_model.md),
and [localhost runbook](https://github.com/howlrs/synapsegit/blob/v0.3.0/deploy/local/README.md)
before evaluating important data.
