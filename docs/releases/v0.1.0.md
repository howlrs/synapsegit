# SynapseGit v0.1.0 — Stage 0 preview

`v0.1.0` is the first tagged SynapseGit preview. It packages the currently
implemented local tools from one tagged commit; it does not declare the Core
protocol frozen or the application production-ready.

## Included binaries

- `synapse` — local Core repository CLI, including validated ingest, Ref
  compare-and-swap, `fsck`, directory export/restore, and the bounded
  three-file creator Pilot.
- `synapse-local` — single-user, read-only creator-session viewer that always
  binds to IPv4 loopback (`127.0.0.1`).

The release archive is
`synapsegit-v0.1.0-x86_64-unknown-linux-gnu.tar.gz`. `SHA256SUMS` lets a
recipient verify the downloaded archive against the checksum published with
the Release, but it is not a cryptographic release signature.

## Install

```bash
curl -LO https://github.com/howlrs/synapsegit/releases/download/v0.1.0/synapsegit-v0.1.0-x86_64-unknown-linux-gnu.tar.gz
curl -LO https://github.com/howlrs/synapsegit/releases/download/v0.1.0/SHA256SUMS
sha256sum --check SHA256SUMS
tar -xzf synapsegit-v0.1.0-x86_64-unknown-linux-gnu.tar.gz

./synapsegit-v0.1.0-x86_64-unknown-linux-gnu/synapse --version
./synapsegit-v0.1.0-x86_64-unknown-linux-gnu/synapse-local --version
```

The archive is built with Rust 1.88.0 on the GitHub-hosted Ubuntu 22.04 image
and requires glibc 2.34 or newer. Only Linux x86_64 GNU is published in this
first release. The source archive generated by GitHub remains available for
source builds on other platforms.
Windows does not currently support the CLI's atomic archive publication path;
macOS has not yet passed the tagged release pipeline.

## Current boundary

- Core v0.1 and `sg-oid-v1` remain Stage 0 drafts.
- `synapse update-ref` is a trusted local operator primitive, not an
  authenticated network authorization boundary.
- The creator Pilot uses fixed process-local identities and caller-supplied AI
  output. It does not invoke a model, prove authorship, or analyze pixels.
- `synapse-local` is a read-only loopback application. It must not be exposed
  through a reverse proxy or treated as a public/multi-user service.
- HTTP/JWT, durable or distributed authorization, organization/quorum/release
  approval, OS sandbox/egress control, and the public cloud implementation are
  not included.

Read the
[security model](https://github.com/howlrs/synapsegit/blob/v0.1.0/docs/security_model.md),
[CLI reference](https://github.com/howlrs/synapsegit/blob/v0.1.0/docs/cli_reference.md),
and
[localhost runbook](https://github.com/howlrs/synapsegit/blob/v0.1.0/deploy/local/README.md)
before using the preview with important data. These tag-pinned links also work
from the `README.md` included in the binary archive.
